# AWS 배포 설명

## 1. EC2를 선택한 이유

1. 강사님과 배웠던 게 기억에 남았고, 가장 익숙했기 때문에
2. 프리티어 무료 활용
3. 자주 쓰이는 방법이기 때문에

## 2. 보안 그룹 설정 이유

0.0.0.0/0으로 모든 IP를 가능하게 만들었는데, 시간에 쫓기고 아직 보안 그룹 설정이 익숙하지 않아
강사님께 배웠던 강의 내용을 떠올리면서 복습하는 차원에서 똑같이 만들어 보았다.

## 3. 서버에서 실행한 명령 흐름

1. EC2 접속
2. Docker 설치하기 (컨테이너로 실행시키기 위해서)
3. Docker Compose 설치 (여러 컨테이너를 편하게 관리하기 위해서)
4. 스왑 메모리 설정 //메모리 부족 현상이 일어났던 부분// 
`sudo dd if=/dev/zero of=/swapfile bs=128M count=16
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile`
5. 실행 (docker-compose up -d : 백그라운드에서 실행해서 터미널을 닫아도 계속 실행되게 하기 위해서)

## 4. 배포 후 접속 방식
### 프론트
1. 브라우저에 http://43.201.26.159:3000 입력
2. 리스트 화면이 정상 표시되는지 확인
3. 데이터 입력 후 등록
4. 새로 등록한 맛집이 표시되는지 확인

### 백
1. 브라우저에 http://43.201.26.159:8080/api/restaurants 입력
2. 맛집 데이터 표시되는지 JSON 형태 확인
