# 프로젝트 설계

## 1. 서비스 설명
사용자가 방문했던 곳을 등록하는 맛집 리스트 서비스이다.

## 2. 전체 흐름
1. 웹에서 맛집 정보(이름, 카테고리, 위치, 별점, 한줄평)을 입력
2. 프론트에서 받은 입력 데이터를 백엔드 api로 전송
3. 백엔드가 MySQL에 데이터를 저장
4. 목록 조회 시 백이 MySQL에서 데이터를 가져와서 응답
5. 프론트가 응답받은 데이터를 화면에 표시

## 3. 프론트엔드 역할
1. 입력 폼 제공
2. fetch
3. 목록 표시
4. Tailwind css

## 4. 백엔드 역할
1. POST, GET
2. 데이터 유효성 검사
3. JPA
4. CORS
