# 프로젝트 회고

## 1. 가장 막혔던 지점

`Failed to load resource: net::ERR_CONNECTION_REFUSEDUnderstand this error
a7458c74c5f3b7d7.js:1 맛집 목록 조회 실패: TypeError: Failed to fetch`

배포 후에 net::ERR_CONNECTION_REFUSED 에러가 발생하며 맛집 목록 조회가 되지 않았다. 
검색 후 원인이 프론트엔드 코드 내 API 주소를 localhost에서 EC2 주소로 수정하지 않았기 때문이란 것을 알게 되었다.
하지만 주소 수정 후 빌드와 푸시를 했음에도 불구하고 동일한 오류가 반복됐다. 원인을 찾던 중 검색을 통해
브라우저 캐시 문제임을 확인했고, 강력 새로고침으로 캐시 삭제 후 접속하니 정상적으로 해결되었다. 
코드 수정뿐만 아니라 배포 환경에서의 캐시 관리도 중요하다는 것을 느끼게 되었다.

## 2. 이해가 부족하다고 느낀 부분

Docker가 제일 어려웠다. Dockerfile에 뭘 써야 하는지, 왜 그렇게 써야 하는지 잘 모르는 상태로 진행하다 보니 openjdk 관련 에러가 났을 때 혼자서는 해결을 못 했다.
결국 AI 도움을 받아서 eclipse-temurin으로 바꿨는데, 왜 openjdk가 안 되는지 그제서야 알게 됐다.
잘 모르는 상태에서 시간에 쫓겨 일단 도구를 쓰고 보자는 식으로 진행한 게 너무 아쉬웠다.
다음에는 설정값이 뭘 의미하는지 먼저 이해하고 적용해야겠다는 생각이 들었다.

## 3. 팀 프로젝트 전에 보완하고 싶은 기술

이번 프로젝트에서 정말 수많은 에러를 접했는데 에러 발생 지점을 정확히 파악하지 못해 정말 많은 시간이 걸렸다..
에러 현상을 효과적으로 설명하지 못하다 보니 간단한 문제도 해결까지 오래 걸리는 경우가 많았다. 이러한 점이 
팀 프로젝트에서 큰 걸림돌이라는 위기감을 느꼈고, 팀 프로젝트에서 이러면 큰일 날 것 같아서 노션에 에러 기록을 남기고
에러 로그를 잘 보는 습관을 들여야 할 것 같다고 느꼈다.

## 4. 혼자 진행하며 느낀 점

혼자 프로젝트를 하면서 나의 부족한 점을 객관적으로 파악할 수 있었다. 수많은 에러를 만나도 대처하는 능력이 부족하다는 점이다.
검색을 하다가 결국 쓰지 말자고 다짐했던 AI를 많이 활용하게 되었다..깊은 반성을 하고 있다. 낮은 트러블슈팅 능력 때문에 
작업이 지체될 때 팀 프로젝트에 대한 걱정이 앞서기도 했다. 하지만 반대로 혼자였기에 불가능했던 소통의 필요성을 많이 느껴서
이번에 혼자 삽질하면서 배운 것들이 팀 프로젝트에서 분명 도움이 될 거라고 생각하고, 
팀원들과 의견을 주고받으며 부족한 부분을 보완해 나간다면 개인 프로젝트 때보다 훨씬 더 많이 성장할 수 있을 것이라 생각한다.
이번 경험을 바탕으로 팀 프로젝트에서는 원활하게 소통하고 프로젝트에 열심히 기여하고 싶다는 생각이 들었다.
